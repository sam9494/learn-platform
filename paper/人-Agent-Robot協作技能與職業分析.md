# 設計「人 + Agent + Robot」協作系統需要的技能 & 橫跨三者的職業

結合 McKinsey 報告內容和產業最新趨勢的完整分析

---

## 一、需要掌握的技能（分四層）

### 第 1 層：基礎技術能力

這是入門門檻，至少需要掌握其中一個方向：

| 領域 | 具體技能 | 說明 |
|------|------|------|
| **程式設計** | Python、C++、JavaScript/TypeScript | Python 是 AI/Robot 開發的通用語言；C++ 用於機器人控制；JS/TS 用於 Agent 前端與 API |
| **AI/ML 基礎** | PyTorch 或 TensorFlow、LLM 原理、Prompt Engineering | 理解模型如何「思考」才能設計有效的人-Agent 協作 |
| **機器人基礎** | ROS2 (Robot Operating System)、感測器整合、控制系統 | 理解機器人如何感知和行動 |
| **Agent 開發** | LangChain/LangGraph、CrewAI、AutoGen、MCP/A2A 協議 | 報告中提到的 Model Context Protocol 和 Agent2Agent 正是讓 Agent 團隊溝通的關鍵協議 |
| **資料架構** | 向量資料庫（Pinecone、Weaviate）、圖資料庫、API 設計 | Agent 需要記憶系統，Robot 需要感知數據管道 |

### 第 2 層：系統設計與整合能力（核心差異化技能）

這是**真正設計三者協作**的關鍵層，也是目前市場上最稀缺的：

| 能力 | 具體內容 | 對應報告洞見 |
|------|------|------|
| **工作流程設計 (Workflow Design)** | 將業務流程拆解為「人做什麼、Agent 做什麼、Robot 做什麼」的能力 | 報告核心論點：重新設計工作流程是釋放 $2.9 兆價值的關鍵 |
| **多 Agent 協調架構 (Multi-Agent Orchestration)** | 設計多個 Agent 如何分工、傳遞上下文、共享記憶、協調決策 | 報告案例中銷售團隊有 5-6 個 Agent 協作，銀行案例每個開發者管理 15-20 個 Agent |
| **人機交互設計 (Human-AI Interaction Design)** | 定義人類何時介入、如何介入、回饋如何流回系統 | Deloitte 研究指出：僅 6% 的組織在此領域領先，但刻意設計人-AI 互動的組織超越 ROI 預期的可能性高出 2 倍 |
| **安全與信任架構** | 設計 Human-in-the-loop 審核點、異常處理流程、偏見檢測 | 報告強調信任和安全是領導者的核心責任 |
| **邊界設計** | 判斷哪些任務交給 Agent、哪些交給 Robot、哪些必須由人完成 | 報告的 SCI 指數正是為了量化這個邊界 |

**Deloitte 的研究特別強調：** 一家電信公司只是加入 AI 工具但不重新設計工作流程，生產力只提升 5%；但將 90% 的部署預算花在**人-AI 互動重新設計**上，則生產力提升了 30%。這說明「設計能力」比「技術能力」更有價值。

### 第 3 層：領域知識 (Domain Expertise)

報告明確指出 **60% 的生產力收益來自行業特定領域**，所以必須深入理解至少一個行業：

| 行業 | 為什麼重要 | 需要理解的工作流程 |
|------|------|------|
| **製造業** | 自動化採用率最高 (31%)，Robot 占比最大 (32%) | 供應鏈管理、品質檢測、生產規劃 |
| **醫療** | 價值 $351B，但需高度人類判斷 | 患者分診、臨床文件、實驗結果解讀 |
| **金融** | Agent 占比最高 (92%)，高度知識密集 | 貸款核保、KYC、風險評估 |
| **專業服務** | 價值 $323B，人-Agent 協作最典型 | 研究綜合、論點建構、客戶諮詢 |
| **零售/物流** | 三者協作最明顯的場景 | 訂單履行、庫存管理、客戶服務 |

### 第 4 層：軟技能（報告反覆強調的「持久技能」）

| 技能 | 為什麼不可替代 |
|------|------|
| **系統思維** | 設計三者協作需要看到全局，不只是單一技術 |
| **變革管理** | 報告案例都顯示：讓人接受新的工作方式是最大挑戰 |
| **溝通與敘事** | 需要向非技術的利益相關者解釋「為什麼這樣分工」 |
| **批判性思維** | 判斷 AI 的輸出是否可靠、Robot 的行為是否安全 |
| **同理心** | 理解人類員工在轉型中的焦慮與需求 |

## 二、橫跨「人 + Agent + Robot」的新興職業

### Tier 1：直接設計三者協作的核心職業

| 職業 | 做什麼 | 需要的技能組合 | 市場狀態 |
|------|------|------|------|
| **AI Agent 協調專家 (AI Agent Orchestration Specialist)** | 設計 Agent 之間以及 Agent 與人的協作流程，定義誰做什麼、何時升級、如何品質控管 | 流程工程 + AI 開發 + 領域知識 + 變革管理 | Eightfold AI 稱之為「2026 年最重要的工作」；有此角色的組織達到 Agent 全面生產力的速度快 65%，員工滿意度高 3 倍 |
| **人-AI 互動設計師 (Human-AI Interaction Designer)** | 設計人類與 AI 系統的互動介面、決策流程、回饋機制 | UX/服務設計 + AI 理解 + 行為心理學 + 資料分析 | Deloitte 2026 報告的核心主題；目前僅 6% 的組織有此能力 |
| **工作流程架構師 (Workflow Architect)** | 分析現有業務流程，重新設計為人-Agent-Robot 協作的新工作流程 | 業務流程管理 (BPM) + AI/自動化 + 產業知識 | 報告的 190+ 工作流程分析正是這個角色的產出 |
| **AI 機器人整合工程師 (AI-Robotics Integration Engineer)** | 讓 AI Agent 的決策能驅動實體 Robot 行動，並確保人類可安全監督 | ROS2 + AI/ML + 感測器融合 + 安全工程 | 報告中太陽能案例（AI 無人機 + AI 巡視車 + AI Agent + 現場技術員）的設計者 |

### Tier 2：在三者交集中工作的延伸職業

| 職業 | 切入角度 | 典型場景 |
|------|------|------|
| **AI 產品經理** | 從產品策略角度定義人-Agent-Robot 的協作方式 | 決定一個 SaaS 產品中哪些功能由 Agent 自動化、哪些保留人類判斷、是否整合物理設備 |
| **數位轉型顧問 / AI 策略顧問** | 幫助企業規劃整體的 AI 轉型路徑 | 對應報告第四章的六大領導者問題，為 C-suite 提供建議 |
| **AI 信任與治理專家** | 確保三者協作中的安全、倫理、合規 | 設計偏見檢測、稽核軌跡、異常升級機制 |
| **AI 培訓師 / 技能轉型設計師** | 幫助員工適應新的人-Agent-Robot 工作模式 | 報告指出企業需要投資持續學習，此角色需求正在飆升 (+90 個職業提及「教學」技能) |
| **Embodied AI 工程師** | 專注於讓 AI 具備物理環境中的感知與行動能力 | 人形機器人、自動駕駛、倉儲機器人的 AI 控制系統 |

### Tier 3：從現有職業轉型的路徑

報告強調「技能鄰接性」——你不需要從零開始，很多現有職業可以透過補充技能切入：

| 現有職業 | 補充什麼技能 | 轉型為 |
|------|------|------|
| **軟體工程師** | + Agent 框架 (LangChain) + 工作流程設計 | Agent 協調專家 / 工作流程架構師 |
| **UX/UI 設計師** | + AI 互動模式 + 行為心理學 | 人-AI 互動設計師 |
| **機器人工程師** | + LLM 整合 + Agent 協議 (MCP/A2A) | AI 機器人整合工程師 |
| **管理顧問/MBA** | + AI 流暢度 + 流程自動化理解 | AI 策略顧問 / 數位轉型顧問 |
| **專案經理/Scrum Master** | + Multi-Agent 架構理解 + AI 工具 | AI 產品經理 / Agent 協調專家 |
| **產業工程師 (IE)** | + AI/ML 基礎 + Robot 整合 | 智慧製造流程設計師 |
| **HR / L&D 專員** | + AI 流暢度 + 技能分析框架 | AI 技能轉型設計師 |

## 三、具體學習路線建議

### 路線 A：偏技術（Agent + Robot 開發者）

```
Step 1: Python + AI/ML 基礎 (3-6 個月)
    → Coursera / Fast.ai / Andrew Ng 課程

Step 2: Agent 開發實戰 (3-6 個月)
    → LangChain / LangGraph / CrewAI
    → 理解 MCP (Model Context Protocol) 和 A2A (Agent2Agent)
    → 自己做一個多 Agent 協作的專案

Step 3: 機器人基礎 (3-6 個月)
    → ROS2 入門
    → 感測器融合 + 電腦視覺基礎
    → 嘗試在模擬環境 (Gazebo/Isaac Sim) 中讓 Agent 控制 Robot

Step 4: 整合實戰 (持續)
    → 設計一個完整的 人-Agent-Robot 工作流程原型
    → 例如：AI Agent 做訂單分析 → Robot 做倉庫揀貨 → 人類做品質審核
```

### 路線 B：偏設計（工作流程架構師 / 互動設計師）

```
Step 1: AI 流暢度建立 (2-3 個月)
    → 深度使用各種 AI 工具 (Claude, GPT, Copilot)
    → 理解 AI 的能力邊界（什麼做得好、什麼做不好）

Step 2: 流程設計方法論 (3-6 個月)
    → Business Process Management (BPM) 基礎
    → 服務設計 (Service Design) + 設計思維
    → 學習 McKinsey 報告中的工作流程分類框架

Step 3: 人-AI 互動設計 (3-6 個月)
    → 研讀 Deloitte 的 Human-AI Interaction Design 框架
    → 學習定義 AI 的角色類型：AI 作為監督者/協作者/教練/思考夥伴
    → 實際設計幾個 Human-in-the-loop 流程

Step 4: 產業深耕 (持續)
    → 選一個行業深入，理解其核心工作流程
    → 用 SCI 思維分析該行業的技能變化
```

### 路線 C：偏管理（AI 策略顧問 / 轉型領導者）

```
Step 1: 建立 AI 素養 (2-3 個月)
    → 理解 Agent、Robot、GenAI 的能力與限制
    → 親自使用並建立直覺

Step 2: 掌握分析框架 (3-6 個月)
    → McKinsey 的七大職業原型 + SCI 指數
    → 190+ 工作流程分類框架
    → Deloitte 的 Macro/Micro 設計框架

Step 3: 變革管理能力 (3-6 個月)
    → 組織變革理論 (Kotter, ADKAR)
    → 如何建立心理安全感
    → 如何設計技能轉型計畫

Step 4: 實戰 (持續)
    → 在自己的組織中主導一個「工作流程重新設計」的小型專案
    → 用報告中的框架進行分析和提案
```

## 四、關鍵洞察總結

1. **最有價值的不是單一技術，而是「跨界整合」的能力。** 能同時理解人、Agent、Robot 三者的人極其稀缺
2. **「設計力」> 「開發力」。** 報告和 Deloitte 研究都指出：技術實施不是瓶頸，工作流程重新設計才是。會寫 Agent 的人很多，能設計人-Agent-Robot 協作流程的人很少
3. **產業知識是護城河。** 報告顯示 60% 的價值在行業特定領域——通用的 AI 技能容易被取代，但「懂金融 + 懂 Agent 設計」的人極為稀有
4. **這是一個全新的領域。** 報告中提到的「AI Agent 協調專家」這個職位在 2023 年幾乎不存在，現在被稱為「2026 年最重要的工作」。先行者有巨大的先發優勢
